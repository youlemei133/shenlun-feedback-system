## Users

### 前台用户
- **身份**：申论考生，备考学习者
- **目标**：对比两种参考答案和两种批改体验，反馈个人偏好
- **场景**：备考学习时，需要快速理解两种方案的差异并做出判断
- **核心任务**：快速对比答案 → 查看批改详情 → 明确选择偏好 → 提交反馈

### 后台管理员
- **身份**：内容配置人员、数据分析人员
- **目标**：配置题目、答案、批改详情，查看统计数据，导出反馈数据
- **场景**：日常工作环境，需要高效完成数据录入和分析
- **核心任务**：录入题目 → 配置答案 → 编辑批改 → 查看统计 → 导出数据

## Brand Personality

### 三个关键词
**清新** | **友好** | **高效**

### 情感目标
传达轻松友好的学习体验，让用户感到这是一个舒适、温暖的备考工具。

### 设计语气
- **清新**：使用柔和的色彩和干净的视觉
- **友好**：温暖的色调和亲切的视觉语言
- **高效**：简洁的布局，服务于效率

```css
--text-xl: 1.25rem;    /* 20px - 副标题 */
--text-2xl: 1.5rem;    /* 24px - 标题 */
--text-3xl: 2rem;      /* 32px - 大标题 */
--text-4xl: 2.5rem;    /* 40px - 页面标题 */
```

#### 行高
```css
--leading-tight: 1.25;   /* 标题、短文本 */
--leading-normal: 1.5;   /* 正文 */
--leading-relaxed: 1.75; /* 长文本、材料阅读 */
```

### 间距规范

#### 间距系统
```css
--space-xs: 4px;   /* 极小间距：图标与文字、小标签内部 */
--space-sm: 8px;   /* 小间距：同组元素间、按钮内边距 */
--space-md: 16px;  /* 中间距：卡片内边距、段落间距 */
--space-lg: 24px;  /* 大间距：区块间距、section 间距 */
--space-xl: 32px;  /* 超大间距：页面头部、重要区块 */
--space-2xl: 48px; /* 巨大间距：页面分隔、主要区块 */
--space-3xl: 64px; /* 特大间距：页面顶部/底部 */
```

#### 视觉节奏
- **紧密集群**：相关元素使用 `--space-sm` 到 `--space-md`
- **宽松分隔**：独立区块使用 `--space-lg` 到 `--space-xl`
- **页面节奏**：顶部留白 `--space-xl`，底部留白 `--space-2xl`

## Design Principles

### 1. 信息优先
设计服务于信息传达，不干扰用户判断答案和批改的差异。视觉元素应帮助用户快速理解内容，而非分散注意力。

### 2. 效率至上
用户能快速完成对比和反馈流程：
- 减少不必要步骤
- 关键操作一键可达
- 状态反馈即时呈现
- 滚动和导航流畅

### 3. 严谨克制
- 避免花哨动画（可保留必要的过渡动画）
- 不使用过度装饰元素
- 色彩运用克制，主色用于强调，不大面积堆砌
- 图标和徽章适度使用

### 4. 状态清晰
用户需要明确知道当前状态：
- **选中状态**：边框高亮 + 背景变化
- **对比状态**：A/B 标签清晰区分
- **反馈进度**：步骤指示器或进度条
- **加载状态**：明确的加载指示

### 5. 可访问性
满足 WCAG AA 基础要求：
- 颜色对比度 ≥ 4.5:1（正文）、≥ 3:1（大文本）
- 支持键盘导航（Tab 键顺序合理）
- 支持屏幕阅读器（语义化标签、ARIA 属性）
- 焦点状态可见（focus-visible）
- 支持 `prefers-reduced-motion` 减少动画

## Components Reference

### 卡片（Card）
用于承载内容区块，圆角 16px，轻微阴影。

### 答案对比卡片（Answer Card）
A/B 版本区分，选中时边框高亮，内部包含来源标签和答案内容。

### 批改详情弹窗（Review Modal）
底部滑入式弹窗，固定标题栏，可滚动内容区，支持折叠面板。

### 统计图表（Charts）
使用 Chart.js，配色与整体风格协调，避免过于鲜艳的颜色。

### 富文本编辑器（Rich Editor）
后台配置使用，工具栏简洁，支持加粗、颜色标注、列表等基础功能。

## Responsive Design

### 设计理念
移动优先设计，核心体验针对手机端优化，同时适配平板和桌面。

### 断点系统

| 断点 | 范围 | 目标设备 |
|------|------|----------|
| xs | < 360px | 小屏手机 |
| sm | 360px - 480px | 标准手机 |
| md | 481px - 640px | 大屏手机 |
| lg | 641px - 768px | 平板竖屏 |
| xl | 769px - 1024px | 平板横屏/小桌面 |
| 2xl | 1025px+ | 桌面 |

### 响应式策略

#### 手机端 (< 768px)
- 单列布局
- 全宽卡片
- 底部固定按钮
- 触控优化（44px 最小触控区域）
- 分屏页面上下滚动

#### 平板端 (768px - 1024px)
- 部分双列布局
- 价值卡片横向排列
- 分屏页面左右布局
- 增大间距和字号

#### 桌面端 (1025px+)
- 居中显示，最大宽度 480px-520px
- 模拟手机界面（带阴影和圆角）
- 弹窗居中显示
- 优化鼠标悬停交互

### 横屏处理
手机横屏时采用紧凑布局：
- 价值卡片横向排列
- 减少垂直间距
- 保持核心功能可用

### 打印支持
- 隐藏交互元素（按钮、弹窗）
- 每页一个视图
- 保留核心内容
